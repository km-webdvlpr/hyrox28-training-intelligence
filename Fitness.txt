This prompt is a complete specification. Do not ask clarifying questions. Build the full app as described.

Project Name:
Hyrox28 Training Intelligence

Purpose:
This is a single-user personal performance tracking and analytics web app for Hyrox-style training. It is not a social app, not a multi-user system, and not a commercial product. It is designed for one user to log workouts, track adherence, monitor performance, and review trends via a dashboard.

Scope constraints:

* No authentication system
* No login screen
* No multi-user support
* No payments
* No social/community features
* App should open directly to dashboard
* Single-user mode assumed

Data persistence:

* Use Supabase (latest JS client)
* Use public anon key
* No auth wall
* Single user assumed at all times

Frontend stack:

* React
* TypeScript
* Vite
* TailwindCSS
* react-router-dom v6
* react-hook-form
* zod (for validation)
* date-fns (for date handling)
* Recharts (for charts)
* clsx + tailwind-merge (cn utility)

State management:

* useState + useContext only (no Redux)
* Keep state simple and local-first

---

Routing structure:

* / → Dashboard
* /log → Log Workout
* /history → Workout History
* /analytics → Analytics

---

Responsive layout:

* Mobile-first
* sm: 640px
* Dashboard cards:

  * 1 column mobile
  * 2 column tablet
  * 3 column desktop

* id

workout_exercises:

* id

* sets

body_metrics

Log Workout Form:

Use react-hook-form + zod.

Fields:

* date
* program_block
* program_week
* program_day
* title
* workout_type
* status
* duration_minutes
* rpe
* notes

Exercise rows must support dynamic conditional inputs:

Rules:

* strength → show sets, reps, weight
* run → show distance_m, time_seconds
* row → show distance_m, time_seconds, calories
* ski → show distance_m, time_seconds, calories
* hyrox movements → show distance or reps depending on type
* mobility → minimal fields

Do NOT show all fields at once.
Inputs must dynamically show/hide based on category.
